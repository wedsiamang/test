## ガベージコレクタ
- JVMではガベージコレクタが任意のタイミングで稼働し、プログラムが使用しなくなったメモリ領域を検出し解放する。
- ガベージコレクタはスレッドで稼働しており、優先度は低く設定され、基本的にはメモリが圧迫されると稼働する。
- プログラマが強制的に実行させることはできないが、以下にてオブジェクトをガベージコレクタの対象にすることは可能。
  ①nullを代入する
  ②参照変数を他のオブジェクトに割り当てる
    オブジェクトを参照していた変数に、新たな参照先オブジェクトを割り当てることで、元のオブジェクトがどこからも参照されなくなったら、そのオブジェクトはガベージコレクタの対象となる。
