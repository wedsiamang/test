# コンパイル
**javac**コマンドでコンパイルする。コンパイルが成功すると、ソースファイルを保存している同じディレクトリ内に.class拡張子のついたファイルが自動生成される。
.classファイルはクラスファイルと呼ばれ、JVMが解釈、実行できる形式のファイル。
コンパイルに成功したら、**java**コマンドでクラスファイルを実行する。.classはつけない。

>従来)
>javac ソースファイル名.java
>java クラスファイル名
```
javac Main.java
java Main
```
>11以降)
>可能になった、クラスファイルを使用せず実行する
classpath上に実行されるクラス名と同じ名前のクラスファイルがある場合エラーになるため、カレントディレクトリにクラスファイルがない状態で実行する必要がある。
```
java Main.java
```
#### 配列宣言
無名配列
```
int[]ary;
ary=new int[]{};
```
参照型のためnullを許容。
```
Double[]ary={null,1.0};
```
コンパイルエラー
```
int[]ary=new int[3]{};
```
>[!WARNING]
>大括弧にサイズを記載する場合は、中括弧を書いてはいけない。矛盾が出てしまうため。

#### メンバ変数とローカル変数の初期化コンパイル
メンバ変数は明示的に初期化しなくてもコンパイラが自動的に初期化するが、ローカル変数は初期化されない、コンパイルエラーとなる。

#### オーバーロードメソッドに対する互換性
- new int[3]と互換性があるのは、Object型。
- プリミティブ配列とObject配列は互換性がない。
- int型配列とlong型配列は互換性がない。
