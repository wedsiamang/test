# ラムダ式
ラムダ式から参照されるローカル変数は final,または事実上のfinalである必要があり、finalを付与しなくてもfinalであるため、値の変更は不可。

構文)
```
(実装するメソッドの引数) -> {処理};
```
### 左辺
- 省略なし
```
(String str) ->
```
- 型推論による型の省略
  インタフェースの宣言時に引数の型は決定しているため、実装時に明記しなくても推論できる
```
(str)->
```
- ()の省略
引数が1つの場合は()の省略が可能。引数がない場合や複数の引数がある場合は省略できない。
```
str->
```
- 引数がない場合
```
()->
```
### 右辺
- 省略なし
```
{ return "Hello" + str; }
```
-{}の省略、return の省略(処理が1文の場合は、{}の省略が可能、{}を省略する場合、さらに return も省略可能
```
"Hello" + str;
```
