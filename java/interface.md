# インタフェース
- インタフェースで宣言した抽象メソッドは、修飾子を省略していた場合、暗黙で public abstract が付与される。
- 実装クラスでオーバーライドする場合は、publicを付与する必要がある。
- オーバーライドする場合、変数名は異なっていても良い。
- インタフェースを実現したクラスで具象メソッドを提供する場合は、アクセス修飾子を変更することはできない。
- インスタンス化できない
- 公開するためのものなので、publicなメンバを定義する
- 抽象メソッドを定義できる
- デフォルトメソッドとstaticメソッドとして実装を定義できる
- デフォルトメソッドとstaticメソッドから利用されることを想定してprivateな具象メソッドを定義できる
- 定数は定義できるが、フィールドは定義できない

#### オーバーライド
- super() 実行中に呼ばれるオーバーライドメソッドでは、子クラスのフィールドはまだ初期化されていないため、その時点ではデフォルト値となる
- フィールドの初期化はコンストラクタ実行の後に行われる
- コンストラクタ呼び出しの順番は必ず親→子。型が親でも子でも同じ。
- static,private,finalはオーバーライドできない。
